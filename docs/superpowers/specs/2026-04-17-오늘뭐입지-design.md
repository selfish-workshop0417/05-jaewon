## 결정 사항 요약

| 항목 | 결정 |
|------|------|
| 운세 데이터 소스 | Gemini API (무료 티어) |
| 색상 자동 추출 | Gemini API (이미지 분석) |
| 데이터 저장 | 로컬스토리지 (MVP) → 서버 DB (추후) |
| 기술 스택 | Next.js |
| 주 사용 환경 | 모바일 우선, 반응형 |
| 메인 화면 우선순위 | 날씨+기온 → 운세+행운색 → 추천 코디 |
| 행운색 없을 때 폴백 | ① 액세서리 대체 → ② 날씨 기준 추천 |

## 핵심 로직: 추천 알고리즘

1. Gemini API에서 오늘 행운색 수신
2. 로컬스토리지 옷장에서 해당 색상 필터링
3. 날씨 기온에 맞는 카테고리(아우터/긴팔/반팔 등) 교차 필터
4. 결과 있으면 → 추천 출력
5. 결과 없으면 → 액세서리 카테고리에서 행운색 검색
6. 그것도 없으면 → 행운색 조건 제거, 날씨 기준만으로 추천
