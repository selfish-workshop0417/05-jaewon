# 오늘 뭐 입지? — PRD

## 1. 서비스 개요
아침마다 날씨 확인과 옷 선택이 번거로운 개인을 위한 도구형 웹서비스. 오늘의 운세(오하아사 or 사주)와 날씨를 자동으로 불러와 행운색 기반 옷을 추천해준다. 내 옷장에 등록한 옷 또는 쇼핑몰에서 저장한 이미지 중에서 골라준다.

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)
- 오늘의 운세 자동 조회 (오하아사 or 사주 API 연동)
- 날씨 자동 조회 (위치 기반 날씨 API)
- 행운색 추출 → 해당 색상 옷 매칭
- 내 옷장 등록 (직접 업로드 or 쇼핑몰 이미지 URL 저장)
- MVP: 운세+날씨 기반 추천 + 옷장 등록

상세는 `planning/` 폴더 참조. [작성 예정]

## 4. 디자인 (Design)
아침에 빠르게 확인하는 서비스이므로 심플하고 직관적인 UI. 오늘의 행운색이 메인 화면에 크게 보이고, 추천 옷이 카드 형태로 노출되는 흐름.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
타겟: 아침이 바쁘고 옷 고르는 게 귀찮은 나. 핵심 여정: 앱 열기 → 오늘의 운세+날씨 확인 → 추천 옷 확인 → 입고 나가기.

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)
개인 도구이므로 초기엔 마케팅 불필요. 추후 지인 공유 or SNS 바이럴 가능성 검토.

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)
개인 사용 목적으로 시작. 반응 좋으면 멀티유저 확장 및 구독 모델 검토.

상세는 `strategy/` 폴더 참조. [작성 예정]

## 8. 역할 배정 (가상)

| 팀 | 담당 영역 | 작업 폴더 |
|----|---------|----------|
| PM (본인) | 전체 기획·의사결정 | 루트 |
| 기획 팀 | 기능 정의, MVP 범위 | planning/ |
| 디자인 팀 | UI/UX, 비주얼 | design/ |
| CX 팀 | 페르소나, 여정맵, FAQ | cx/ |
| 마케팅 팀 | 런칭, 채널 | marketing/ |
| 전략 팀 | 수익 모델, 성장 | strategy/ |
