# MVP 범위

## MVP에 포함 (지금 당장 만들 것)

| 기능 | 이유 |
|------|------|
| 날씨 자동 조회 | 핵심 서비스의 기반, 없으면 의미 없음 |
| 운세 조회 + 행운색 추출 | 서비스의 핵심 차별점 |
| 행운색 기반 옷 추천 | 날씨+운세를 합치는 핵심 로직 |
| 내 옷장 등록 (이미지 업로드) | 추천의 재료, 없으면 추천 불가 |

## MVP 제외 (나중에 추가할 것)

| 기능 | 이유 |
|------|------|
| 쇼핑몰 URL로 옷 저장 | 이미지 크롤링 기술 필요, 2단계에서 추가 |
| 색상 자동 감지 | AI 이미지 분석 필요, 초기엔 수동 태그로 대체 |
| 멀티유저 | 개인용으로 시작, 반응 보고 결정 |
| 알림 (아침 푸시) | 웹앱 기본 기능 먼저, 이후 PWA or 앱으로 확장 |
| 사주 기반 운세 | 오하아사 먼저 연동, 사주는 2단계 |

---

## MVP 완성 기준

- [ ] 날씨 API 연동해서 오늘 날씨 보여주기
- [ ] 오하아사 운세 + 행운색 보여주기
- [ ] 옷장에 옷 등록 (이미지 업로드 + 색상 수동 태그)
- [ ] 행운색 + 날씨 기반 옷 추천 1건 이상 노출
